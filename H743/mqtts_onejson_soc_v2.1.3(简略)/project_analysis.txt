# OneNET MQTT 物模型 SDK 项目分析报告

## 1. 项目概述

本 SDK 是基于 MQTT 协议和 OneJSON 数据格式开发的物联网设备端软件开发工具包，专门用于连接中移物联网 OneNET 平台。它实现了设备身份认证、物模型数据交互（属性、事件、服务）、网关子设备管理以及安全传输等核心功能。

## 2. 业务功能介绍

SDK 提供的核心业务功能包括：

- **设备连接与管理**：支持 MQTT/MQTTS 连接、设备登录/登出及维护。
- **物模型交互 (Thing Model)**：支持属性读写、事件上报、服务调用及期望属性同步。
- **网关与子设备管理**：支持子设备拓扑管理及代理数据交互。
- **数据优化**：提供打包上报和带时间戳的历史数据批量上报。
- **机卡协同**：集成蜂窝/非蜂窝设备信息的协同上报功能。

## 3. 底层适配接口 (HAL/PAL)

移植 SDK 需实现以下底层接口：

### 3.1 网络接口 (plat_tcp.h)

- `plat_tcp_connect`: 建立 TCP 连接。
- `plat_tcp_send`: 发送 TCP 数据。
- `plat_tcp_recv`: 接收 TCP 数据。
- `plat_tcp_disconnect`: 断开 TCP 连接。

### 3.2 时间接口 (plat_time.h)

- 提供系统毫秒/秒级计数、延时及倒计时器支持。

### 3.3 系统工具 (plat_osl.h)

- 提供内存管理、字符串/内存操作及随机数生成。

## 4. 最简应用方案 (设备远程控制)

若实现基本的远程控制（如开关灯）：

1. **必需模块**: `3rd/paho-mqtt`, `3rd/cJSON`, `onenet/tm` 核心 API 及协议封装。

2. 实现步骤

	:

	- 在 `tm_user.c` 定义控制属性（如 `light_switch`）。
	- 在 `tm_prop_light_switch_wr_cb` 中编写 GPIO 控制逻辑。
	- 在主循环调用 `tm_step()` 处理下行指令。

## 5. AT 指令模块（如 ESP8266）适配

### 5.1 无网络协议栈平台

SDK 内部集成了协议逻辑，底层的 `plat_tcp.h` 仅需简单的字节流抽象，不强制要求 LwIP 等原生协议栈。

### 5.2 如何基于 AT 指令实现底层？

开发者需通过 UART 发送 AT 指令来模拟 TCP 行为：

1. **连接 (`plat_tcp_connect`)**: 发送 `AT+CIPSTART` 并解析 `CONNECT OK`。
2. **发送 (`plat_tcp_send`)**: 发送 `AT+CIPSEND`，等待 `>` 后发送原始数据。
3. 接收 (`plat_tcp_recv`):
	- **核心点**: 必须使用 **UART 中断 + 环形缓冲区** 捕获模块异步推送的 `+IPD` 数据。
	- 在接口中剥离协议头，将纯净数据返回给 SDK。
4. **断开 (`plat_tcp_disconnect`)**: 发送 `AT+CIPCLOSE`。
5. **系统节拍**: 裸机平台需提供 SysTick 保证 `plat_time.h` 的超时机制正常运行。
